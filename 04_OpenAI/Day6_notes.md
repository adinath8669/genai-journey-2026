## 💡 What I Understood

### 1. What is an API?

An API (Application Programming Interface) is a way for two systems to communicate with each other.

In my case, my Python code sends a request to the OpenAI API, and the API returns a response from the AI model.

So basically:

> My code → API → AI Model → Response → My code

---

### 2. What is OpenAI Client?

The OpenAI client is a Python object that helps us connect to the OpenAI API easily.

Instead of handling complex HTTP requests manually, the client handles everything for us like:

* Sending prompts
* Receiving responses
* Managing authentication using API keys

---

### 3. What are Model Parameters?

Model parameters are settings we pass to control how the AI behaves.

Important parameters include:

* **model** → which AI model to use (example: gpt-4, gpt-3.5)
* **temperature** → controls randomness
* **max_tokens** → limits response length
* **top_p** → controls diversity of output

These parameters help us customize the AI response.

---

### 4. System vs User Message

In chat-based models, messages are divided into roles:

* **System message** → sets behavior of the AI (rules, personality, instructions)
* **User message** → actual input/question from the user
* **Assistant message** → response generated by AI

Example:

System:

> You are a helpful AI assistant.

User:

> What is AI?

Assistant:

> AI is ...

---

### 5. How Chatbot Loop Works

A chatbot loop means continuously taking input from the user and sending it to the AI model.

Steps:

1. Take user input
2. Send input to OpenAI API
3. Get response from model
4. Display response
5. Repeat until user exits

This loop makes the chatbot interactive like ChatGPT.
